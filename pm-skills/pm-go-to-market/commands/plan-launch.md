# Plan Launch

Create a complete GTM plan for a product launch.

## Step 1: Define Launch Scope

If not provided, ask:
- What is launching? (New product / major feature / expansion to new market)
- What is the target customer for this launch?
- What is the launch date?
- What's the primary goal? (Awareness / Activation / Revenue / Retention)

## Step 2: Define ICP and Targeting

Activate the **ideal-customer-profile** skill.
Define the ICP for this launch. Identify the beachhead segment to target first.

## Step 3: Design GTM Strategy

Activate the **gtm-strategy** skill.
Select the primary GTM motion. Define channel plan, positioning, and launch phases.

## Step 4: Build the Launch Checklist

Generate a complete launch readiness checklist:

**Product Ready:**
- [ ] Feature fully tested on staging
- [ ] Feature flags configured for % rollout
- [ ] Analytics events instrumented and validated
- [ ] Error monitoring in place

**Marketing Ready:**
- [ ] Launch email drafted and tested
- [ ] Blog post or announcement written
- [ ] Social content prepared
- [ ] Landing page or product page updated

**Sales & CS Ready:**
- [ ] Sales team briefed with talking points
- [ ] Competitive battlecard updated
- [ ] CS team trained on new feature
- [ ] FAQ document created

**Internal Ready:**
- [ ] All stakeholders notified with launch date
- [ ] Rollback plan documented
- [ ] On-call schedule confirmed for launch week
- [ ] Launch success metrics defined

## Step 5: Launch Metrics Dashboard

Define what to measure in the first 30 days:
- Day 1: [Immediate metric]
- Week 1: [Short-term metric]
- Month 1: [30-day metric]
- Leading indicators: [Signals that predict long-term success]

## Step 6: Deliver GTM Plan

One-page launch plan:
- Product: [What's launching]
- Target: [ICP + beachhead]
- Date: [Launch date]
- Primary motion: [GTM motion]
- Channels: [Channel list with owners]
- Success metrics: [KPIs by timeframe]
- Launch phases: [Phase 1 → 2 → 3]
- Risks: [Top 3 with mitigations]
