# Pricing

Build a comprehensive pricing strategy for a product.

## Step 1: Gather Inputs

If not provided, ask:
- What is the product?
- Is it B2B or B2C?
- Who are the target segments?
- What do competitors charge?
- What value does the product deliver (time saved, revenue generated, cost reduced)?
- Is there an existing pricing model to evaluate?

## Step 2: Run Pricing Analysis

Activate the **pricing-strategy** skill.

Evaluate candidate pricing models. Build the value anchor (quantify value delivered, set price as % of value). Design tier structure. Apply behavioral pricing principles.

## Step 3: Build the Pricing Page Structure

Recommend:
- Number of tiers (typically 3)
- Names that signal value (e.g., Starter / Pro / Enterprise, not Basic / Plus / Premium)
- Feature allocation across tiers
- Which features gate each upgrade
- Annual vs. monthly pricing and discount

## Step 4: Write Pricing Rationale

For each tier, explain:
- Target customer persona
- Why the price is set at this level
- What the upgrade trigger is (what makes a customer move up)

## Step 5: Pricing Test Plan

Recommend an A/B testing approach:
- What variable to test first (price point / number of tiers / feature split)
- How long to run the test
- What metric determines a winner (conversion rate / ARPU / LTV)

Deliver the full pricing recommendation as a table plus a narrative rationale.
