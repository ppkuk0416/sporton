# Brainstorm OKRs

Use when: the user wants to set quarterly or annual OKRs, align team goals with company strategy, or review and improve existing OKRs.

Trigger: "OKRs", "objectives and key results", "quarterly goals", "team OKRs", "set goals", "goal setting"

## Framework (Google OKR Methodology)

### Objectives — Quality Criteria
A great Objective is:
- **Inspiring**: Creates energy and direction
- **Qualitative**: Not a number — a direction
- **Time-bound**: Clearly scoped to a period (quarter/year)
- **Ambitious**: Feels slightly uncomfortable — 60–70% achievement = success
- **Action-oriented**: Starts with a verb

Bad: "Improve user experience"
Good: "Become the most intuitive data tool in our category"

### Key Results — Quality Criteria
A great Key Result is:
- **Measurable**: Has a clear number and baseline
- **Outcome-focused**: Measures change, not activity
- **2–5 per Objective**: Too few = incomplete, too many = unfocused
- **Aggressive but realistic**: Should require real effort to achieve

Bad: "Launch 3 new features"
Good: "Increase 30-day retention from 42% to 58%"

### OKR Structure

```
Objective: [Inspiring, qualitative direction]

KR1: [Metric] from [baseline] to [target] by [date]
     Confidence: [%] | Owner: [name]

KR2: [Metric] from [baseline] to [target] by [date]
     Confidence: [%] | Owner: [name]

KR3: [Metric] from [baseline] to [target] by [date]
     Confidence: [%] | Owner: [name]

Initiatives: [Key projects or bets that will move these KRs]
```

### OKR Levels

**Company OKRs** (annual): Set direction for the whole organization
**Team OKRs** (quarterly): Contribute to 1–2 company OKRs
**Individual OKRs** (quarterly, optional): Personal growth + team contributions

Best practice: 60–70% of team OKRs should link to company OKRs; 30–40% can be self-directed improvements.

### OKR Anti-Patterns to Avoid

- **Sandbagging**: Setting easy targets to guarantee 100% — defeats the purpose
- **Task lists as KRs**: "Publish 10 blog posts" is an output, not an outcome
- **Too many OKRs**: Max 3 objectives, 4 KRs each
- **No baseline**: Can't measure progress without knowing where you started
- **Annual KRs only**: Need quarterly checkpoints to adjust

### Confidence Tracking

Each KR should have a weekly confidence score (0–100%):
- 70%+ = on track
- 40–70% = at risk, needs attention
- < 40% = off track, escalate immediately

## Output Format

Generate OKRs in the structure above. For each team/company level:
1. 2–3 Objectives
2. 3–4 Key Results per Objective
3. Confidence score and owner for each KR
4. 2–3 suggested initiatives per Objective
5. Alignment map: How team OKRs connect to company OKRs
6. Anti-pattern check: Flag any KR that violates the quality criteria
