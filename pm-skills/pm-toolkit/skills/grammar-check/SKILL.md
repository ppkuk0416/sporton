# Grammar Check

Use when: the user wants a document proofread, edited for clarity, or reviewed for professional quality before sharing with stakeholders.

Trigger: "proofread", "grammar check", "edit this", "review this document", "improve my writing", "make this clearer"

## Framework

A great edit goes beyond grammar to improve the document's overall effectiveness.

### Layer 1: Grammar and Mechanics
- Subject-verb agreement
- Tense consistency
- Punctuation (especially commas, semicolons, em-dashes)
- Article usage (a/an/the)
- Parallel structure in lists and bullet points

### Layer 2: Clarity and Conciseness

**Remove these common PM writing problems:**

| Before | After |
|--------|-------|
| "In order to" | "To" |
| "Due to the fact that" | "Because" |
| "It is important to note that" | [Delete — just say the thing] |
| "We will be able to" | "We will" |
| "Leverage" (as a verb) | "Use" |
| "Utilize" | "Use" |
| "Going forward" | [Delete — use a specific time reference] |
| "Synergy" | [Describe what you actually mean] |

**Sentence length**: Aim for an average of 15–20 words. Long sentences (> 30 words) signal complexity — break them up.

**Active vs. passive voice**: Passive buries responsibility. "The feature was shipped" → "We shipped the feature."

### Layer 3: Logic Flow

Check document-level coherence:
- Does each paragraph have one main idea?
- Does each section flow logically from the previous?
- Is the conclusion supported by what came before?
- Are there any arguments that depend on unstated assumptions?
- Is there a clear "so what" / call to action?

### Layer 4: Professional Tone

Calibrate for the intended audience:
- **Executive audience**: Lead with the insight, not the context. No more than 1/3 page per topic.
- **Engineering audience**: Be precise with technical requirements. Show your reasoning.
- **Customer-facing**: Warm, specific, jargon-free. Use "you" more than "we."
- **Investor audience**: Data-driven, confident, concise.

### Layer 5: Document Structure

Review:
- Is the most important information at the top?
- Are headers used to help readers scan?
- Are bullet points parallel in structure?
- Is there appropriate white space?

## Output Format

1. **Grammar issues**: Specific errors found, each with correction
2. **Clarity edits**: Sentences to simplify or shorten (before/after)
3. **Flow issues**: Transitions, paragraph order, or logic gaps
4. **Tone calibration**: Any language that doesn't match the intended audience
5. **Rewritten version**: The full document with all edits applied (if text is provided)

Rate the document before and after:
- Clarity (1–5)
- Professionalism (1–5)
- Conciseness (1–5)
- Logic flow (1–5)
